# proxysss Local Verification Lab

Copy this folder to a runtime directory (for example `D:\Server\proxysss_dir`) and use it to verify HTTP, HTTPS, TCP, UDP, FTP passthrough, and WebDAV through proxysss.

## Layout

```text
proxysss_dir/
  proxysss.yaml      # gateway config
  gateway.ts         # script routes for echo upstreams
  start-lab.ps1      # start echo backends + proxysss
  stop-lab.ps1       # stop background jobs
  verify-lab.ps1     # curl-style smoke checks
  webdav/            # WebDAV root
  certs/             # TLS material (generated by init or ACME)
  plugins/           # optional plugins
```

## Quick Start (Windows)

```powershell
cd D:\Server\proxysss_dir
.\start-lab.ps1
.\verify-lab.ps1
```

## Echo Backends

| Service | Upstream | Via proxysss |
| --- | --- | --- |
| HTTP echo | `127.0.0.1:8081` | `http://127.0.0.1/echo` |
| TCP echo | `127.0.0.1:7001` | `tcp://0.0.0.0:26379` |
| UDP echo | `127.0.0.1:8101` | `udp://0.0.0.0:2053` |
| FTP passthrough | `127.0.0.1:2121` (TCP echo) | `ftp://0.0.0.0:21` |
| WebDAV | local `./webdav` | `http://127.0.0.1/dav` |

## HTTPS

### Local (self-signed, immediate)

Default `proxysss.yaml` uses `tls.mode: self_signed`. Test with:

```powershell
curl.exe -k https://127.0.0.1/
curl.exe -k https://127.0.0.1/echo
```

### Production (ACME)

1. Install `acme.sh` and ensure your domain points to this host.
2. Copy `proxysss.acme.example.yaml` settings into `proxysss.yaml`.
3. Set real `email` and `domains`.
4. Run `proxysss run --config .\proxysss.yaml` on ports 80/443.

## Stop

```powershell
.\stop-lab.ps1
```
