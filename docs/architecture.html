proxysss

Interactive architecture map — one Rust binary, nginx-class edge duties.

Big Picture

Single-binary edge gateway

proxysss terminates protocols, matches routes, enforces policy, proxies to upstream pools, and exposes observability — optional TypeScript plugins extend business logic without sitting on every hot-path byte.

Core (Rust)

HTTP/HTTPS/H2/H3, low-latency TCP, UDP/KCP-style streams, MQTT/IoT, FTP, WebDAV, static files, AI reverse proxy, cache, compression, rate limits.

Config

One YAML file (proxysss.yaml) is the source of truth with hot reload fingerprinting.

Runtime

Active health, watchdog heartbeat metrics, and supervised critical background loops.

HTTP Path

Reverse proxy & load balancing

Requests hit plain/TLS/H3 listeners, pass access control and rate limits, then match static sites, WebDAV, native AI proxy routes, domain routes, reverse-proxy routes, or script hooks.

Domain-first routing: services.domain_routes groups host/path upstream pools with per-route compression, cache, and rate limits.
AI API routes: services.ai_proxy handles New API, sub2api, and OpenAI-compatible upstreams with path rewrite and provider metadata before generic HTTP routes.
Algorithms: rendezvous (default), round-robin, least-connections, source-hash, weighted — with sticky affinity and active health probes.
Resilience: retry policy, passive quarantine, manual upstream drain via admin API, forwarding headers (X-Forwarded-*, Forwarded).
services:
  domain_routes:
    - name: api
      domains: [api.example.com]
      upstream: http://127.0.0.1:8080
      upstreams: [http://127.0.0.1:8081]
      upstream_weights:
        http://127.0.0.1:8080: 1
        http://127.0.0.1:8081: 3
Certificates

TLS & automatic HTTPS

Default public TLS listens on 0.0.0.0:443. Managed ACME uses built-in HTTP-01 and TLS-ALPN-01 without external tooling.

Built-in path: http.tls.auto_https / acme_managed — ordinary hostname certificates.
Wildcard boundary: built-in Use http.tls.mode: acme_managed + http.tls.acme.challenge: dns01 with embedded providers (cloudflare, aliyun_cn, aliyun_intl, tencent, volcengine, aws, azure, google, or manual). No acme.sh/certbot required. Legacy acme_dns_external remains for non-built-in providers only.
On-demand TLS: http.tls.on_demand issues managed ACME certs on first SNI hit when allow patterns match (optional ask_url gate).
SNI: Multi-cert selection via http.tls.certificates and per-domain manual SSL on routes.
Long-lived HTTP

WebSocket & gRPC-over-H2

WebSocket upgrades tunnel bidirectionally after a successful 101 Switching Protocols handshake to compatible upstreams.

WebSocket: Route to ws:// or wss:// upstreams; hop-by-hop headers are stripped appropriately.
gRPC: transparent HTTP/2 reverse proxy forwards application/grpc payloads when upstreams speak HTTP/2 — no special directive required.
Realtime Streams

TCP, UDP, and KCP-style fleets

Direct stream listeners keep game, voice, device, and AI tool bridge payloads transparent while bounding connection churn in the Rust hot path.

TCP

tcp.listeners[].nodelay defaults true; connect_timeout_ms bounds upstream dial latency.

UDP/KCP

session_ttl_secs refreshes on each datagram; idle associations are pruned automatically.

Scale guard

max_associations caps per-listener client mappings for churn-heavy realtime fleets.

MQTT/IoT

MQTT TCP, MQTT TLS passthrough/SNI, MQTT over WebSocket, and CoAP-style UDP stay transparent at the edge.

tcp:
  listeners:
    - name: game-tcp
      bind: 0.0.0.0:7000
      protocol: game_tcp
      nodelay: true
      connect_timeout_ms: 3000
udp:
  listeners:
    - name: game-kcp
      bind: 0.0.0.0:7001
      protocol: kcp
      session_ttl_secs: 180
      max_associations: 262144
Shaping

Rate limiting

Shared-zone policies protect HTTP routes and TCP stream listeners using the same algorithm implementations.

HTTP

services.rate_limit.http — fixed window, token bucket, or leaky bucket; optional connection caps per zone key.

Streams

services.rate_limit.stream — limits new TCP connections per client IP in a shared zone.

Keys

Scope by remote IP, Host header, or arbitrary header value for API-key style quotas.

Health: load_balance.active_health supports HTTP, TCP, and opt-in UDP probes; UDP can require a response or record a successful send.
Watchdog: runtime.watchdog restarts failed critical loops when enabled and exposes heartbeat/failure counters on /metrics.
Response policy

Cache & compression

Global defaults in services.response_policy; routes override compression and cache. Zones can be memory-backed with optional disk files.

Behaviors: bypass, respect_origin, override, no_cache with edge/browser TTL and optional CDN-Cache-Control.
Cache keys: key_prefix and vary_headers (e.g. Accept-Encoding) build variant-aware keys.
Stale policies: stale_while_revalidate_secs background refresh; stale_if_error_secs serves stale on upstream failure.
PURGE: Active invalidation when allow_purge: true.
Compression: Negotiates zstd, brotli, or gzip based on Accept-Encoding, content-type allowlist, and minimum body size.
Control + data

FTP gateway

When services.ftp.native_control=true, proxysss provides nginx ftp module directive-level parity: control-channel proxying, passive (PASV/EPSV) and active (PORT/EPRT) data rewriting, policy hooks, and structured lifecycle logs.

Policy: command_allow / command_deny plus transfer_allow / transfer_deny and per-user user_policies.
Observability: Structured logs for control commands, transfer hooks, and data-channel bridge lifecycle.
Directive mapping: bind, proxy_pass/upstream, pasv_address/public_ip, passive port range, timeout, login, and rate knobs map into services.ftp.
Control plane

Admin & agent API

Loopback dashboard on 127.0.0.1:7777 exposes health, stats, upstream drain, plugin load/unload, and automation endpoints.

Read-only defaults: enable_write_ops=false until operators enable cluster automation.
Upserts: Bearer-token POST /v1/domain-routes/upsert (and TCP/UDP/listener siblings) persist into YAML and reload in-process.
Inspection CLI: proxysss config explain, capabilities, nginx-parity, reload-plan.
Hardening

Security model

SSRF guards: Blocked upstream host lists and mutation validation.
HTTP/1 smuggling: Ambiguous requests rejected when enabled.
Admin auth: Rate-limited failed logins with lockout windows.
Secrets: Redacted in proxysss config show; atomic YAML writes on automation.
In-cluster

Kubernetes ingress-style mode

When kubernetes.enabled=true, service DNS mappings expand into services.domain_routes on each load/reload — useful for in-cluster edge registration without hand-editing every pod hostname.

kubernetes:
  enabled: true
  namespace: prod
  cluster_domain: cluster.local
  mappings:
    - name: api
      service: api-svc
      port: 8080
      domains: [api.example.com]
Extensions

Embedded TypeScript runtime

Scripts run inside the binary via QuickJS with in-process TypeScript stripping (SWC). Hooks like access, routeHttp, and plugin modules are optional — the default gateway path avoids script calls on hot paths.

Main script: gateway.ts exported default plugin object.
Auto plugins: plugins/*.ts with optional <name>.plugin.yaml sidecars.
Not supported: Node/Deno/tsc, npm module graphs, or external runtimes.
Operations

Hot reload boundaries

Hot: Routes, policies, scripts, plugins, most services.* values, upstream health overrides.
Restart: Listener binds, TLS mode, admin bind, FTP enable/bind, logging sink paths/format.
Inspect: proxysss config reload-plan lists what changes need a process restart vs in-process reload.
